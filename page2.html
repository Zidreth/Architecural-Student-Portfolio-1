~Interoperability~
Position Readonly Textbox
This project analyzes the interoperability between Blender and Rhino3D, using Blender 4.2.3 and Rhino 8. Comparison charts were developed to see the differences in 3d structure when exporting models from Blender space to Rhino space. Multiple different export options and import settings allow for variation, this variation is analyzed to determine which export option is best when working on a project.

A manipulated open plane, closed object, and character model were used in the project. Featured differences in the chart include object type, polygon count, visuals, and file size.
Accompanying the charts, a 3D printed model, low-poly paper model, and a photogrammetry scan of the paper model

The main takeaway about interoperability that I learned from this exercise was the extent to the manipulations you can get of an object just by simply changing settings or the export option. Additionally, OBJ and FBX are the optimal imports when going between Rhino and Blender as they are lightweight are both supported by the programs with the most accurate data. When it comes to NURBS and Mesh modelling, I learned that NURBS mainly consists of geometry created by curves, and meshes are geometry created by points. Mesh geometry is typically easier to handle computationally, as in the file size differences, all the meshes were relatively low, whereas the SubD was significantly larger. When it comes to using the information I have learned for future projects I can determine which tool is correct for the job. For example, if I wanted to create a scene I would probably opt for mesh objects as they will not bog down the processor as much. It is also important to know which export options work for which programs when it comes to inter-program collaboration with other professionals.
